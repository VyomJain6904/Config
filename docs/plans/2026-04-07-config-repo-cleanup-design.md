# Config Repo Cleanup Design

**Objective**
- Remove non-config application state, cache files, logs, and session artifacts from this dotfiles repository so commits stay focused on real configuration.

**Current State**
- The largest non-config footprint is in `Config-Arch/Code`, `Config-Arch/Antigravity`, and `Config-Arch/obsidian`.
- These folders contain runtime browser/electron data (`Cache`, `Code Cache`, `CachedData`, `logs`, crash dumps, storage DB files) rather than portable user configuration.
- Existing meaningful tracked changes are in `Config-Arch/i3/config`, `Config-Arch/opencode/opencode.json`, and `Config-Arch/opencode/package.json`.

**Selected Approach**
- Approach A: purge app-state aggressively and add ignore rules to prevent re-introduction.

**Scope**
- Delete app-state directories:
  - `Config-Arch/Code`
  - `Config-Arch/Antigravity`
  - `Config-Arch/obsidian`
- Add a repository-level `.gitignore` for common runtime artifacts:
  - cache directories and files
  - logs and crash dumps
  - storage/session DB artifacts
  - Python bytecode cache (`__pycache__`, `*.pyc`)

**Out of Scope**
- Modifying install scripts or changing the config copy mappings.
- Editing valid config directories such as `i3`, `polybar`, `nvim`, `rofi`, `yazi`, `ghostty`, etc.

**Risk and Mitigation**
- Risk: deleting app-state can remove local session history.
- Mitigation: these paths are untracked runtime artifacts in this repo context and can be regenerated by applications; portable configs are retained elsewhere.

**Validation**
- Verify the three app-state directories are removed.
- Verify `git status` no longer lists cache/log flood and only lists meaningful config changes plus `.gitignore`.
- Verify top-level disk usage is substantially reduced for `Config-Arch`.

**Expected Result**
- Clean repository tree suitable for committing and pushing without accidental bloat.
